How it works

Most "file sharing" sites upload your file to their server, store it, and hand the other person a download link. Beam doesn't. When you pick a file, your browser opens a direct, encrypted connection to the other person's browser using WebRTC — the same peer-to-peer technology behind video calls — and streams the file straight across. The only thing that passes through a server is a tiny handshake that introduces the two browsers to each other. After that, the bytes flow directly between you.

Why send a file this way?

It's private by construction: there's no copy of your file sitting on someone's cloud waiting to be leaked, scanned, or kept after a "link expires." It's fast, because the file takes the shortest path — directly between the two devices — instead of a round trip up to a server and back down. And there's no real size cap, since you're not paying for anyone's storage. It's the closest thing the open web has to AirDrop, except it works between any two browsers, on any two platforms.

The honest caveats

Both browser tabs need to stay open during the transfer, since the file flows live between them. The connection uses public STUN servers to find a direct path, which works on the large majority of home and office networks; a small number of very locked-down corporate networks block direct peer connections, in which case the handshake won't complete. And the handshake (just the connection details, never your file) is relayed so the two browsers can find each other.

Is my file uploaded anywhere?

No. The file goes directly from your browser to the other person's browser over a WebRTC data channel. Only a small connection handshake is relayed to introduce the two browsers. Your file bytes never touch a server.

Is it encrypted?

Yes. WebRTC data channels are encrypted in transit with DTLS by default, so the file is encrypted on the wire between the two browsers.

How big a file can I send?

There's no fixed limit. The file is streamed in chunks peer-to-peer, so the practical ceiling is your connection speed and both tabs staying open, not a storage quota.

Do we need accounts?

No sign-up for either side. One person picks a file and shares the link or QR; the other opens it and the transfer starts.

Does it work between a phone and a laptop?

Yes. Scan the QR code on a phone, or open the link on any device. It works across platforms because it's just two web browsers talking to each other.
